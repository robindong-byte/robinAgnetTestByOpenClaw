📚 理论部分（45分钟）
1. 为什么要"切分"文档？
- 一篇文章几千字，直接整篇 Embedding 效果差（信息太稀释）
- 切成小块（Chunk），每块 200-500 token，检索精度更高
- 切分策略：
- 固定长度：每 N 个字符切一块（简单粗暴）
- 按段落/句子：保留语义完整性（推荐）
- 滑动窗口：相邻块有重叠（避免关键信息被截断）
2. Chunk 重叠（Overlap）的意义
[chunk1: 第1-200字]
[chunk2: 第150-350字] ← 50字重叠
[chunk3: 第300-500字] ← 50字重叠
重叠确保跨块的关键句子不会丢失。
3. 完整 RAG 流水线
